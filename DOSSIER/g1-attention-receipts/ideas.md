- JSONL sibling absence — CLOSED, and larger than the gap framed it. Worse, `forge-vscode` and `forge-rsync-mv` write JSONL but are NOT in the registry. The registry and the JSONL corpus are misaligned in both directions; the dual-receipt law is only partially realized. Source: `receipts.json` projection + `fd -e jsonl $HOME/Library/Logs`.

Source: `ls /etc/newsyslog.d`, `rg rotate|newsyslog|logrotate modules/`.

Source: live `zellij --session <s> subscribe -p terminal_0 -s 15 -f json` and `list-panes --all --json` on 0.44.3.

### Ecosystem-side (ecosystem.md GAPS)

An offline machine needs no extension download at first query; `read_json*` is always available. Source: live `duckdb -c "SELECT install_mode FROM duckdb_extensions() WHERE extension_name='json'"`.

Admission is a fixed-output derivation over the SIGNED, notarized release binary (macOS 13+), never a Nix source build — a rebuild forfeits the TCC identity (see the alerter admissibility row below). Source: `nix eval nixpkgs#alerter.version` (absent).
- `terminal-notifier` nixpkgs state — CLOSED. Present at `nixpkgs#terminal-notifier` v2.0.0 (`pkgs/by-name/te/terminal-notifier/package.nix`); the estate's daily receipt corpus proves the live posting path works on the current OS. Source: `nix eval nixpkgs#terminal-notifier.version` → `2.0.0`.
- `ntfy` nixpkgs identity — CLOSED. The Go client/server is `nixpkgs#ntfy-sh` v2.24.0 (near the 2.26.0 upstream head); the bare `nixpkgs#ntfy` attr is a different, older 2.7.1 package (not the binwiederhier client). Any estate row targets `ntfy-sh` (see the ntfy estate-fit row below for the receive-side story). Source: `nix eval nixpkgs#ntfy-sh.version` → `2.24.0`, `nixpkgs#ntfy.version` → `2.7.1`.
- `noti` nixpkgs — CLOSED. Present at `nixpkgs#noti` v3.8.0. The non-adoption verdict stands (the collector's tty+process fold is richer for session attention). Source: `nix eval nixpkgs#noti.version` → `3.8.0`.

- `zjframes.wasm` surface — CLOSED, and out of scope for attention. It is a separate background plugin that toggles pane-frame borders only (renders no status text), for users running a non-zjstatus bar; three string keys (`hide_frame_for_single_pane`, `hide_frame_except_for_search`, `hide_frame_except_for_fullscreen`), zellij 0.41+, loaded via `load_plugins` and requiring global `pane_frames` disabled. Not a notification or attention renderer — no F11 relevance. Source: https://github.com/dj95/zjstatus/wiki/6---zjframes.
- zjstatus `{pipe}` key roster — CLOSED. Exactly two keys: `pipe_NAME_format` (`{output}` = last message) and `pipe_NAME_rendermode` (`static` ignores directives, `dynamic` renders `#[..]` found inside the pipe content, `raw` passes ANSI through). No interval/cwd/env keys — those belong to `{command}`, not `{pipe}`. The estate's `pipe_agents`/`pipe_quota` run `dynamic`, so their cells already honor `#[..]` styling. Source: https://github.com/dj95/zjstatus/wiki/4-%E2%80%90-Widgets.
- `{notifications}` styling + broadcast — CLOSED, with a correction. Config keys: `notification_format_unread` (`{message}` placeholder), `notification_format_no_notifications`, `notification_show_interval` (default 5s). The notify PAYLOAD does NOT accept `#[..]` directives — the source substitutes the raw body into the format's content and applies only the FORMAT part's style; `#[..]` in a payload renders literally (no `rendermode`/`dynamic` path, unlike `{pipe}`). Per-message urgency color is therefore impossible via the payload; it must live in `notification_format_unread` or route through a distinct renderer. A destination-less `zellij pipe "zjstatus::notify::MSG"` broadcasts to EVERY running plugin instance — it reaches both the `zjstatus` and `zjstatus-hints` bars; `--plugin <url>` targets one. Sources: https://raw.githubusercontent.com/dj95/zjstatus/main/src/widgets/notification.rs ; https://zellij.dev/documentation/plugin-pipes.
- `alerter` admissibility — CLOSED (admission path resolved). The released binary is code-signed AND notarized by Apple (macOS 13+), a bare binary (not a `.app`), posting under its own `fr.vjeantet.alerter` bundle identity; `--sender` overrides. `--reply`/`--actions` ride the same signed path. Estate constraint: a Nix `swift build` from source produces an UNSIGNED binary that macOS 15+ refuses for TCC — so admission requires a fixed-output derivation over the SIGNED release artifact (fetch the notarized binary unmodified), never a source build. Sources: https://raw.githubusercontent.com/vjeantet/alerter/master/README.md ; https://raw.githubusercontent.com/vjeantet/alerter/master/docs/release-notes-26.2.md.
- `UNUserNotificationCenter` ceiling — CLOSED. The framework is bundle-scoped throughout: `current()` returns "your app's notification center", categories register per-app, authorization is per-app. A bare CLI with no bundle identity cannot obtain authorization; a signed binary carrying an embedded Info.plist bundle id (alerter's mechanism) can. There is no API surface for a bundle-less process — this is why alerter, not a raw first-party CLI, is the pragmatic answer-channel. Sources: https://developer.apple.com/documentation/usernotifications/unusernotificationcenter ; https://developer.apple.com/documentation/usernotifications/unnotificationaction.
- `ntfy` estate fit — CLOSED. No native macOS receive app: the Mac receives via the web PWA (Dock-installable) or CLI `ntfy subscribe` (config `~/Library/Application Support/ntfy/client.yml`); native subscriber apps exist only for iOS/Android. Self-hosting is a single statically-linked Go binary (server+client+CLI); current server release v2.26.0 (2026-07-09). Verdict: ntfy is the cross-device/phone reach channel, not a mac-local renderer. Sources: https://docs.ntfy.sh/subscribe/phone/ ; https://docs.ntfy.sh/install/.

## [02]-[F01_ATTENTION_VOCABULARY]

- Superset row with a `source` discriminator, not a rename. The feed today is single-source (hook events only) with a closed `event` domain {Stop, Notification, SessionEnd}; every other emitter writes to its own `.receipts.{log,jsonl}`. Thesis: a `source` key (`hook`, `tunnel`, `drift`, `bell`, `deck`) plus a small `urgency` ordinal added at the emit fold turns four private receipt streams into one attention feed the collector already knows how to fold. Verified by: the collector's fold already dedupes by normalized tty and gates on `event == "Notification"` within a one-hour window (`mcp-launchers.nix:945-957`); adding a discriminator is a fold predicate, not a new owner. Unlocks: per-source notification policy (the collector already carries `notifyPolicy.needsInput` + `minIntervalSec` — extend to a policy row per `source`). Anchor: `receipts.nix` emit fold + `forge-agents` collector.
- Urgency as an ordinal the renderers sort on, derived not stored. Thesis: rather than have each emitter assert urgency, derive it at the fold from fields already present — `result=error`/`state=failed` → high, `event=Notification` → needs-input, `event=Stop` → idle. Verified by: the live census shows every terminal emitter carries `result` and the tunnel carries `state=failed|ok`; `forge-receipts --failures` already filters on `result`/`status` (`browsers.nix:233-246`). Unlocks: one urgency ladder every renderer (zjstatus cell, toast, `{notifications}`) reads without per-emitter cooperation. Anchor: the fold in `receipts.nix`; policy rows beside `forge-agents`.
- Fold the drift agent and tunnel supervisor into the attention feed as first-class emitters. Thesis: the drift agent (`deployed=drift`, TSV-only) and tunnel supervisor (`state=failed`) are exactly "something needs you" events that never reach the attention surface today — they die in per-kind `.log` files. Verified by: `nix-drift.receipts.log` carries `deployed=drift` and `tunnel.receipts.jsonl` carries `state=failed` live; the collector publishes to every live zellij session already (`mcp-launchers.nix:1084-1085`). Unlocks: "estate drifted" and "tunnel flapped" render in the same bar cell as agent attention. Anchor: `forge-agents collect` fold; drift/tunnel emitters gain a `source`-tagged feed append.

## [03]-[F02_RECEIPTS_ANALYTICS]

- The query plane is a two-format reader, not a JSONL glob. Decisive finding: the registry enumerates `.log` paths and twelve of eighteen kinds have no JSONL sibling — a plane that globs `*.jsonl` silently drops drift, accept, maintenance, mcp, parity, path-doctor, and the rest. Thesis: `forge-receipts` verbs read JSONL where it exists and parse the TSV `k=v` grain (`split_part`/regex) where it does not, or the registry gains a `format` column and the reader dispatches on it. Verified by: registry rows are `{emitter, kind, path}` today (`receipts.json` live); DuckDB reads both — `read_json_auto` for the six JSONL kinds, `read_csv(..., sep='\t')` + `k=v` split for the twelve TSV kinds. Unlocks: a query plane that actually covers the estate instead of a sixth of it. Anchor: `browsers.nix` receipt rows + registry projection; DuckDB overlay (json statically linked).
- Per-kind typed projections that `CAST` at read. Thesis: verbs like failures-last-24h, redeploy build-time trend, tunnel flap count, acceptance pass/warn/fail deltas — each a fixed SQL projection over one kind, casting the JSON-supertype numerics back to typed columns. Verified by: `union_by_name` collapses `build_s`/`diff_lines` to `JSON` across drifted files (live census), so `CAST(build_s AS DOUBLE)` per-kind is mandatory; `filename` virtual column (DuckDB ≥1.3.0) preserves source identity across a glob. Unlocks: "redeploy is getting slower" and "the tunnel flapped 40 times overnight" as one verb each, not log spelunking. Anchor: DuckDB projections as `forge-receipts --verb` rows; `receipts.nix` grammar.
- Registry-completion pass as a prerequisite, surfaced as a verb. Thesis: `forge-vscode`/`forge-rsync-mv` emit JSONL but aren't registered, and twelve registered kinds lack JSONL — a `forge-receipts audit` verb that diffs registry rows against on-disk `.log`/`.jsonl` reality makes the drift queryable and self-correcting. Verified by: the live misalignment (18 registered, 6 JSONL, 2 unregistered-with-JSONL) is measurable today with `fd` + the registry projection. Unlocks: the dual-receipt law becomes enforceable, not aspirational. Anchor: registry seam (`browsers.nix:66-133`); a coverage verb beside `forge-receipts`.
- Cross-kind attention timeline as a single query. Thesis: `union_by_name` over the whole registry (both formats normalized to `ts, surface, source, urgency, result|state`) yields one chronological "what happened to the estate" table the F07 dashboards (G2) render. Verified by: `read_json([...], union_by_name=true, filename=true)` composes multi-source schema drift with provenance (live-tested); `viddy` floating panels already render receipt queries in-terminal (`ops.nix:34-40`). Unlocks: the F02 plane feeds F07 without a second data path. Anchor: DuckDB corpus reader; `zellij/ops.nix` floating watch rows; ripple to G2 F07.

Thesis: F11 is one pipe-publish added to the collector's existing per-session loop — the same rise the desktop notifier already gates on drives an in-bar transient toast. Verified by: the collector computes the rise (`prev_needs` vs current, throttled by `minIntervalSec`, gated by `notifyPolicy.needsInput`) and already loops `zellij --session "$s" pipe "zjstatus::pipe::…"` over every live session (`mcp-launchers.nix:1084-1085`, transition logic at `:1090-1100`); `{notifications}` is verified on the pinned 0.23.0 line (capabilities.md:42). Unlocks: attention visibility while macOS notifications are muted — one more renderer on the F01 stream, zero new event owner. Anchor: `mcp-launchers.nix` collector loop; `zellij/config.nix` bars gain `{notifications}`.

Correction from this wave: `{notifications}` renders the payload literally (no `#[..]` path), and `notification_format_unread` styles every notification identically. Thesis: since per-message color is impossible, the collector encodes urgency as a leading glyph/prefix in the broadcast body (`!!` high, `?` needs-input) and `notification_format_unread` carries one fixed style; genuine per-urgency color routes through the styled `pipe_agents` cell instead, which DOES honor `#[..]` under `rendermode dynamic`. Verified by: notify payload is literal (`notification.rs`), the `pipe_agents` cell runs `dynamic` and honors directives (this wave). Thesis: a destination-less `zjstatus::notify::` reaches both `zjstatus` and `zjstatus-hints`; F11 either adds `{notifications}` to one bar only (top) and lets broadcast be benign on the hint bar, or targets `--plugin` for a single instance. Verified by: broadcast-to-all-instances confirmed (`zellij/plugin-pipes`); the estate renders two instances (`layouts/default.nix:31-47`). Unlocks: deliberate placement instead of accidental double-toast. Anchor: `zellij/config.nix` bar regions.

Thesis: where `terminal-notifier` posts a fire-and-forget banner, `alerter --reply`/`--actions --json` turns a needs-input rise into a blocking question whose answer returns as typed JSON — the agent's "which option?" becomes a notification the operator answers without switching windows. Verified by: alerter is signed+notarized, posts under `fr.vjeantet.alerter`, and `--reply`/`--actions --json` ride that path (this wave); the notification class split (async banner vs blocking Q&A) is the documented pattern (ecosystem.md:46). Admission is a fixed-output derivation over the notarized binary — NEVER a Nix source build, which forfeits the TCC identity (this wave). Unlocks: the attention fabric gains a return path — answers, not just alerts — beside the existing `terminal-notifier` post. Anchor: a new overlay row fetching the signed alerter; `forge-agents` desktop arm (`mcp-launchers.nix:1090-1100`) gains a policy row selecting notifier vs alerter by urgency.

Thesis: F01's per-source policy rows select the renderer by interaction contract — idle `Stop` → silent cell; `Notification` rise → `terminal-notifier` replaceable banner; a decision-shaped event → `alerter` blocking Q&A; failure → both banner and `{notifications}`. Unlocks: one policy table maps (source, urgency) → renderer. Anchor: `forge-agents` notify policy; F01 vocabulary.
- `ntfy` as the cross-device reach tier, publish-only from the estate. Thesis: a needs-input rise that the operator is away-from-Mac to see publishes to a self-hosted `ntfy` topic with a `Click` deep link back to `forge-agents focus`; the Mac stays the local renderer, `ntfy` is the phone reach. Verified by: `ntfy publish` carries `Click`/`Actions`/`Priority` (ecosystem.md:34); self-host is a single Go binary (this wave); no native mac receiver exists, so ntfy is strictly the remote tier (this wave). Unlocks: attention reaches the phone without a second event owner — one more renderer on the F01 stream, custody via a Doppler auth row on the maghz VPS. Anchor: `forge-agents` desktop arm; maghz VPS service plane (ntfy server); Doppler auth row.
- Concert: one attention fold, four renderers, one query plane. Thesis: F01 (superset feed) + F02 (two-format query plane) + F10 (`peek` context) + F11 (`{notifications}`) compose into one stream the collector already owns — hooks/tunnel/drift/bell publish rows; zjstatus cell, `{notifications}` toast, `terminal-notifier`/`alerter` desktop, and `ntfy` remote render them by policy; `peek` supplies click-context; the query plane makes the whole stream historical. Verified by: the collector is already the convergence point (hook-feed + process lanes + zjstatus cells + desktop post, `mcp-launchers.nix:938-1100`); every renderer above is a verified local or registry-derived capability. Unlocks: "what needs me" is one command; "what needed me last week" is one query. Anchor: `forge-agents` collector as the single fold; F02 registry-completion as the historical spine; ripple to G2 F07 dashboards.

## [GAPS]

- Multi-pane `subscribe` interleaving, `pane_closed` emission on session death vs pane close, backpressure toward a busy producing pane, and NDJSON cadence under heavy output remain unprobed — the initial-frame envelope is verified, the streaming dynamics are not (deferred to avoid stressing live panes; the next wave runs a probe against a purpose-spawned throwaway session).
- `{notifications}` render proof against a crowded `format_right`: interaction with `format_hide_on_overlength`/`format_precedence` when `pipe_agents`+`pipe_quota`+`session` already fill the region, and whether `notification_show_interval` auto-hide competes with the standing `pipe_agents` cell for the same row width.
- `forge-agents focus` failure modes when the zellij session died or the pane id was recycled between notification post and click — unobserved (a live probe requires killing a session, disallowed by estate law); the next wave reasons the fallback from the `focus` resolver code rather than probing.
- `alerter` live TCC behavior once installed from the signed release artifact into a Nix profile: whether the notarized `fr.vjeantet.alerter` identity survives a fixed-output derivation's store placement, and whether the first `--reply` triggers the one-time notification-permission grant cleanly under the estate's noninteractive posture.
- The F01 superset-row schema is designed but unbuilt: which of `source`/`urgency`/`pane_ref` become stored keys versus fold-derived values, and whether the TSV-only emitters (drift, accept, maintenance) gain a JSONL sibling or the query plane parses their `k=v` grain — an F01/F02 boundary the implementer rules.
- Registry-completion mechanics: whether `forge-vscode`/`forge-rsync-mv` (JSONL, unregistered) get registry rows and the twelve TSV-only kinds get a `format` column, versus a reader that dispatches on on-disk sibling presence — decides whether the query plane is registry-driven or filesystem-driven.
