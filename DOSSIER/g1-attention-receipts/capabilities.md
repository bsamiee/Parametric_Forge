# G1 Attention And Receipts Capability Dossier

Estate-side truth for F01/F02/F10/F11 (`docs/FRONTIER.md:9-37`): what the live Forge runtime owns, spelled exactly, with the unexploited channels named. External candidate tools, their option spellings, and composable patterns live in `ecosystem.md`.

## [01]-[ATTENTION_FABRIC]

- The attention seam is live end to end: `.claude/hooks/agent-attention.sh` appends JSONL rows keyed `ts`, `event`, `session_id`, `cwd`, `term`, `wezterm_pane`, `zellij_session`, `zellij_pane`, `tty`; `forge-agents collect` folds them; `forge-agents focus` routes them; `zjstatus` renders exact pipe names from `apps/zellij/config.nix`. Source: `docs/atlas/interconnection.md:68`.
- `forge-agents` owns four state surfaces: cache `$XDG_STATE_HOME/forge/agent-state.json`, feed `$XDG_STATE_HOME/forge/agent-attention.jsonl`, lane cache `$XDG_CACHE_HOME/forge/agent-lanes.json`, receipts `$HOME/Library/Logs/forge-agents.receipts.log`. Source: `modules/home/programs/shell-tools/mcp-launchers.nix:876-880`.
- The collector folds process lanes, hook-feed rows, and quota snapshots into one `forge-agents/v1` cache; a session counts as needing input only when its latest feed event is `Notification`, the row sits inside the one-hour window, and its recorded tty still hosts an idle `claude` lane; the fold dedupes by normalized tty. Source: `modules/home/programs/shell-tools/mcp-launchers.nix:938-957`.
- Feed admission is torn-line tolerant by design: `[inputs | fromjson? | select(type == "object")]` because the feed is live-appended and rotated lock-free. Source: `modules/home/programs/shell-tools/mcp-launchers.nix:945-956`.
- The collector publishes bar cells into every live Zellij session: `zellij --session "$s" pipe "zjstatus::pipe::pipe_agents::$agents_cell"` and `zellij --session "$s" pipe "zjstatus::pipe::pipe_quota::$quota_cell"`. Source: `modules/home/programs/shell-tools/mcp-launchers.nix:1084-1085`.
- The desktop arm posts only on a `needs_input` rise, groups under `forge-agents`, and routes clicks with `terminal-notifier ... -group forge-agents -execute "${profileBin}/forge-agents focus"`; `osascript` notification clicks opened Script Editor — the scar that forced `terminal-notifier`. Sources: `modules/home/programs/shell-tools/mcp-launchers.nix:1090-1100`; `docs/atlas/scars.md:120`.
- The notifier binary is a read-only Darwin-gated option resolving to `${pkgs.terminal-notifier}/bin/terminal-notifier` and empty elsewhere; unguarded interpolation throws at linux eval. Sources: `modules/home/programs/shell-tools/bundle-apps.nix:30`; `docs/atlas/scars.md:89`.
- `forge-agents focus` resolves the latest attention row, focuses by `zellij --session "$zs" action focus-pane-id "$zp"`, resolves the hosting terminal app from tty ancestry, and emits a focus receipt. Source: `modules/home/programs/shell-tools/mcp-launchers.nix:1227`.
- WezTerm's bell arm is already structured attention: `audible_bell = "Disabled"`; the `bell` event always writes a receipt and toasts via `window:toast_notification` only when the bell is outside the focused view. Sources: `modules/home/programs/apps/wezterm/default.nix:258`; `modules/home/programs/apps/wezterm/events.lua:52-58`.
- WezTerm deck receipts write the dual envelope natively: TSV append to `rows.receipts_log` plus a JSONL sibling via `:gsub("%.log$", ".jsonl")`; the guarded-sync block toasts and writes a blocked receipt. Source: `modules/home/programs/apps/wezterm/deck.lua:75-83`, `modules/home/programs/apps/wezterm/deck.lua:298`.

## [02]-[RECEIPTS_PLANE]

- The shared receipt fold accepts either a `k=v` TSV row or a one-line JSON object, appends the TSV row, and appends a JSONL sibling at `${receipt_log%.log}.jsonl`; TSV values pass `tonumber? // .` before JSONL emission. Source: `modules/home/programs/shell-tools/receipts.nix:8-26`.
- The dual-receipt law: mutating rails append both TSV and JSONL with identical envelope keys, `ts` and `surface` always present, `result` on terminal receipts, `state` on transition receipts, numerics as JSON numbers; human-only receipts and per-command envelope dialects are rejected forms. Source: `docs/standards/nix-doctrine.md:141-142`.
- The receipt registry declares twelve static kinds — `redeploy`, `maintenance`, `drift`, `orphan-sweep`, `activation-sweep`, `accept`, `browse`, `workspace`, `wezterm`, `zellij`, `mcp`, `agents` — plus `tunnel-${name}` rows derived from `config.forge.ssh.hosts`; the roster is the `forge.registers.receiptSources` option projected to `~/.config/forge/registers/receipts.json`. Source: `modules/home/programs/shell-tools/browsers.nix:66-133`, `modules/home/programs/shell-tools/browsers.nix:537`, `modules/home/programs/shell-tools/browsers.nix:595`.
- `forge-receipts` exposes `--kind K`, `--since ISO|Nh|Nd`, `--failures`, `--limit N`, `--json|--tsv|--fzf|--follow`, and `--pick kind@ts`; failures filter on `result`/`status` fields, and browser faults survive as `result=error` rows visible to `--failures`. Source: `modules/home/programs/shell-tools/browsers.nix:233-246`, `modules/home/programs/shell-tools/browsers.nix:449`.
- The plane already renders in-terminal: `forge-zellij` watch rows run `viddy --interval 30 -- forge-receipts --failures --limit 15` and `viddy --interval 60 -- forge-receipts --kind drift --limit 5` as floating panels. Source: `modules/home/programs/apps/zellij/ops.nix:34-40`.
- The F02 query plane needs zero new packages: DuckDB is installed at the profile (`/etc/profiles/per-user/*/bin/duckdb`, verified live) and pinned `1.5.4` in the overlay manifest; Harlequin ships with a DuckDB default profile as the SQL front door. Sources: `overlays/manifest.nix:18`; `modules/home/programs/languages/db-tools.nix:17-34`.

## [03]-[PANE_CAPTURE_SUBSTRATE]

- `zellij subscribe` is verified real on the installed 0.44.3 via `zellij subscribe --help`: `zellij [--session <NAME>] subscribe [OPTIONS] --pane-id...` with `-p, --pane-id <PANE_ID>...` repeatable (`terminal_1`, `plugin_2`, or bare `N` equivalent to `terminal_N`), `-s, --scrollback [<N>]` (bare flag delivers all scrollback, `N` delivers the last N lines), `-f, --format <raw|json>` (default `raw`), and `--ansi`. Sources: local `zellij subscribe --help` on zellij 0.44.3; https://zellij.dev/documentation/zellij-subscribe.html
- `subscribe` is a top-level CLI verb, not an `action` subverb — it is absent from the cli-actions page by design, and `--session` targets a named session from an outside shell. Source: local `zellij subscribe --help` on zellij 0.44.3.
- JSON mode emits NDJSON: `pane_update` objects carry `pane_id`, `viewport`, `scrollback`, `is_initial`; `pane_closed` objects carry `pane_id`. Source: https://zellij.dev/documentation/zellij-subscribe.html
- `subscribe` appears nowhere in the estate (`rg` over `modules/`) — F10 builds on an untouched channel.
- One-shot capture: `zellij action dump-screen` dumps the focused pane by default, accepts `--path <PATH>` (stdout when omitted), `-f, --full`, `-p, --pane-id`, `-a, --ansi`; `zellij action edit-scrollback` accepts `-p, --pane-id` and `-a, --ansi`. Source: local `zellij action dump-screen --help` and `edit-scrollback --help` on 0.44.3.
- Inventory verbs carry JSON natively: `zellij action list-panes` and `list-tabs` both accept `-a, --all` and `-j, --json` (verified locally); `forge-zellij state` already folds them into a `forge-zellij-state/v1` envelope. Sources: local `--help` on 0.44.3; `modules/home/programs/apps/zellij/ops.nix:248`.
- Focus is pane-id addressed estate-wide: the graph dispatch runs `zellij action focus-pane-id "terminal_${target}"`, and agent-lane rows read from the collector's `agent-lanes.json` projection. Source: `modules/home/programs/apps/zellij/ops.nix:467`, `modules/home/programs/apps/zellij/ops.nix:90`.

## [04]-[ZJSTATUS_RENDERING]

- The estate pins `zjstatus.wasm` from release `v0.23.0` and grants it `ReadApplicationState`, `ChangeApplicationState`, and `RunCommands`. Source: `modules/home/programs/apps/zellij/default.nix:155-156`, `modules/home/programs/apps/zellij/default.nix:110`.
- Layouts render two instances: top bar `zjstatus`, bottom bar `zjstatus-hints`. Source: `modules/home/programs/apps/zellij/layouts/default.nix:31-47`.
- Top-bar regions: `format_left "#[bg=$surface] {tabs}"`, `format_center "{swap_layout}"`, `format_right "{pipe_agents}{pipe_quota}#[bg=$pink,fg=$background,bold] {session} "`, with `pipe_agents_format`/`pipe_quota_format` set to `"{output}"` and both rendermodes `dynamic`; the hints bar renders `{mode}` plus the chord ribbon. No region includes `{notifications}`. Source: `modules/home/programs/apps/zellij/config.nix:428-441`, `modules/home/programs/apps/zellij/config.nix:459-460`.
- The `{notifications}` widget is verified on the pinned line: it requires zellij 0.40+ (estate runs 0.44.3), prints the last message received on the `zjstatus::notify::<message>` pipe, forbids newlines in the message, auto-hides after `notification_show_interval` seconds, and is configured by `notification_format_unread`, `notification_format_no_notifications`, and `notification_show_interval`. Source: https://github.com/dj95/zjstatus/wiki/4-%E2%80%90-Widgets
- `zjstatus::notify::` appears nowhere in the estate (`rg` over `modules/`, `docs/`, `.claude/`) — F11 is one pipe-publish beside the collector's existing `zjstatus::pipe::` calls.

## [GAPS]

- `subscribe` live behavior is unprobed: NDJSON frame cadence under heavy pane output, multi-pane interleaving on one stream, `pane_closed` emission on session death versus pane close, backpressure toward the producing pane, and `--session` targeting from a fully detached shell — the next wave runs a live probe matrix and records observed envelopes.
- `{notifications}` render proof is missing: interaction with `format_hide_on_overlength` and `format_precedence` in a crowded `format_right`, whether notify payloads accept `#[..]` style directives, and whether a destination-less `zjstatus::notify::` broadcast hits both bar instances (`zjstatus` and `zjstatus-hints`) or needs targeted `--plugin` delivery.
- The F01 attention-event vocabulary has no field inventory: today's feed rows carry no `source` or `urgency` key; the candidate emitter roster (tunnel supervisor, drift agents, WezTerm bell arm, hook feed) needs each emitter's current receipt/log shape captured so the unified row schema is derived from real fields, not invented.
- Envelope drift census: per-kind key sets across the twelve-plus registry sources are unmeasured — which kinds emit `result` versus `state`, which carry legacy quoted-string numerics in old TSV-era JSONL rows, and which sources lack a JSONL sibling entirely; a typed DuckDB projection needs this column census per kind.
- `forge-agents focus` failure modes are unobserved: behavior when the zellij session died or the pane id was recycled between notification post and click.
- JSONL corpus lifecycle: rotation and retention of `.jsonl` siblings is undocumented — whether any log rotation truncates the machine side out of sync with the TSV side.
