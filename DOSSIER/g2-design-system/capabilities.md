# [G2_DESIGN_SYSTEM_CAPABILITIES]

Standards, contrast law, type/motion scale systems, token-build tooling, and self-contained HTML primitives that ground F03 (theme owner as design system) and F07 (estate dashboards). The Nix theming/monitoring ecosystem lens lives in `ecosystem.md`; this file owns the standards-and-tooling axis plus the Forge surface those standards measure against.

## [01]-[LOCAL_OWNER_SURFACE]

The design-system substrate Forge already owns — every standard below is measured against these rows, and every adapter lands as an egress over them.

- [THEME_PROJECTOR]: `modules/home/theme.nix` owns `palette`, `derived`, `roles`, `ansi16`, `syntaxScopes`, `icons`, `projections`, `targets` and projects `forge/theme/palette.json`, `forge-dracula.tmTheme`, `base24.yaml`, `palette.html`, `coverage.json` into XDG config. Source: `modules/home/theme.nix:515`, `:724`, `:816`, `:826-840`.
- [COLOR_RECORD]: `mkColor` renders every hex as `{hex, r, g, b, triple, csv, rgba}` from one row, so consumers receive format-specific projections instead of re-parsing hex. This is the pre-built adapter surface a DTCG `$value`/`$extensions` egress folds over. Source: `modules/home/theme.nix:18-31`.
- [COLOR_PROOF]: `palette.html` computes WCAG 2 relative luminance and contrast ratios in-page against base, surface, and overlay landings; `forge-theme-proof` shape-checks `palette.json`, renders terminal color rows, and persists a receipt. Source: `modules/home/theme.nix:567`, `:593`, `:626`, `:640`, `:715`.
- [BASE24_EGRESS]: `base24Slots` maps the palette into `base00`–`base17`; the YAML carries `system = "base24"`, `name = "Forge Dracula"`, `variant = "dark"`, lowercased hex `palette`. The header comment declares this an ecosystem exchange adapter, not the canonical palette owner. Source: `modules/home/theme.nix:531-533`, `:559`.
- [TARGETS_REGISTRY]: `targetsProved` fails evaluation when any non-`system-owned` owner path is missing on disk — the target registry is already the eval-time enrollment/coverage analog of an external theming engine's target API. Source: `modules/home/theme.nix:453-460`.
- [FONT_OWNER]: `modules/home/fonts.nix` owns the family catalog, type roles, fallback chains, per-family metrics, OpenType feature policy, and a manifest checking name-table parity and shaping coverage; metrics already carry `size`, `lineHeights`, `editorLineHeight` (`1.5`), `cssLineHeight` (`140%`). The type-ramp/letter-spacing rows F03 wants land as a metrics row-group here. Source: `modules/home/fonts.nix:7-11`, `:105-126`, `:187-205`.
- [REGISTER_AND_RECEIPT_SEAMS]: `config.forge.theme`/`.fonts`/`.registers.*` are read-only projection hinges; register rows project to `forge/registers/*.json`, theme artifacts to fixed XDG paths (`~/.config/forge/theme/palette.json`, `forge-dracula.tmTheme`). Every `forge-*` kernel persists TSV receipts to `~/Library/Logs/forge-<name>.receipts.log` with JSONL siblings via the shared `append_receipt` fold — the F07 dashboard substrate. Source: `docs/atlas/interconnection.md:64-72`, `modules/home/programs/shell-tools/receipts.nix:7`, `:24`.
- [DASHBOARD_PRECEDENT]: `forge-update-board` proves the observation-only pattern — one row per manifest family folded from existing receipts and local metadata (flake inputs, deploy, MCP pins, Homebrew, tool families) with zero mutation of owner state. Source: `modules/home/programs/shell-tools/forge-tools.nix:1994-2016`.

## [02]-[DTCG_TOKEN_STANDARD]

- [STATUS]: Format Module `2025.10` was published as a Draft Community Group Report on 2026-06-17; its own status text declares it a preview — not a W3C Standard, not on the Standards Track, not an authoritative build target. Treat DTCG as an egress/interop shape, never a required internal representation. Source: https://www.designtokens.org/tr/drafts/format/#status-of-this-document.
- [FILE_SHAPE]: Token files are JSON, media type `application/design-tokens+json`, extensions `.tokens` or `.tokens.json`. Format keys: `$schema`, `$value` (required on a token), `$type`, `$description`, `$extensions`, `$deprecated`. Token/group names must not begin with `$` or contain `{`, `}`, `.`. Source: https://www.designtokens.org/tr/drafts/format/#file-format, https://www.designtokens.org/tr/drafts/format/#design-token.
- [TYPE_SET]: Primitives `color`, `dimension`, `fontFamily`, `fontWeight`, `duration`, `cubicBezier`, `number`; composites `strokeStyle`, `border`, `transition`, `shadow`, `gradient`, `typography`. Every F03 growth axis (type ramp, elevation, motion, density) maps to a type here: type ramp → `dimension`/`typography`, motion → `duration`/`cubicBezier`/`transition`, elevation → `shadow`. Source: https://www.designtokens.org/tr/drafts/format/#types.
- [DIMENSION_MOTION_KEYS]: `dimension`, `duration` (units `ms`, `s`), `cubicBezier` (`[P1x, P1y, P2x, P2y]`, x restricted to `[0,1]`), `transition`. Source: https://www.designtokens.org/tr/drafts/format/#dimension, #duration, #cubic-bezier, #transition.
- [TYPOGRAPHY_KEYS]: `fontFamily`, `fontWeight`, composite `typography` sits beside the primitive family/weight types. Source: https://www.designtokens.org/tr/drafts/format/#font-family, #font-weight, #typography.
- [COLOR_SHAPE]: Color Module `2025.10` — `$type: "color"`, `$value` object with required `colorSpace` + `components`, optional `alpha`, optional 6-digit CSS `hex` fallback, `none` for a missing component. The `hex` fallback maps directly onto `mkColor.hex`; `components` maps onto `{r,g,b}`. Source: https://www.designtokens.org/tr/drafts/color/#color-type, #format.
- [COLOR_SPACES]: `srgb`, `srgb-linear`, `hsl`, `hwb`, `lab`, `lch`, `oklab`, `oklch`, `display-p3`, `a98-rgb`, `prophoto-rgb`, `rec2020`, `xyz-d65`, `xyz-d50`. `oklch` is the shared axis with the html-studio OKLCH-only color law — a palette.json→oklch projection satisfies both the DTCG color shape and the artifact style floor. Source: https://www.designtokens.org/tr/drafts/color/#supported-color-spaces.

## [03]-[CONTRAST_LAW]

- [WCAG22_THRESHOLDS]: AA text contrast `4.5:1` normal / `3:1` large; AAA `7:1` normal / `4.5:1` large. Source: https://www.w3.org/TR/WCAG22/#contrast-minimum, #contrast-enhanced.
- [WCAG22_MATH]: sRGB relative luminance `L = 0.2126·R + 0.7152·G + 0.0722·B`, each channel linearized at the `0.04045` breakpoint with `2.4` exponent; contrast ratio `(L1 + 0.05) / (L2 + 0.05)`, L1 lighter. The math is deterministic over sRGB hex pairs — Forge already computes it in `palette.html`, and `base24Slots` gives every role a hex, so the full role×landing assertion set is enumerable at Nix eval with zero runtime, satisfying the F03 "WCAG-contrast proofs as flake checks" unlock. Source: https://www.w3.org/TR/WCAG22/#dfn-relative-luminance, #dfn-contrast-ratio, `modules/home/theme.nix:593`.
- [APCA_STATUS]: APCA is an emerging readability-contrast algorithm for WCAG 3 work, not a WCAG 2.2 success criterion; it is polarity-sensitive (text/background order matters), so it scores rather than pass/fail-gates. `apca-w3` is the approved public-guideline code (`npm i apca-w3`); Color.js implements version `0.0.98G-4g` via `contrast(color, "APCA")`. Use APCA as an advisory readability score layered over the WCAG gate, never as the gate. Source: https://github.com/Myndex/SAPC-APCA, https://colorjs.io/docs/contrast.html#accessible-perceptual-contrast-algorithm-apca.
- [COLOR_JS_CONTRAST_API]: `contrastWCAG21(color)`, `contrast(color, "WCAG21" | "APCA" | "Lstar" | "DeltaPhi")`; APCA requires the background-receiver-then-text argument order the docs show. One JS API exposes every contrast method a proof board folds. Source: https://colorjs.io/docs/contrast.html.

## [04]-[TYPE_AND_SPACE_SCALE]

- [MODULAR_SCALE]: A modular scale is repeated multiplication of a base size by a ratio; Utopia's calculator exposes named ratios `1.067`–`2`, separate min/max viewport widths, min/max font sizes, and min/max type scales. Source: https://utopia.fyi/type/calculator/, https://utopia.fyi/blog/css-modular-scales/.
- [FLUID_OUTPUT]: Fluid interpolation between two modular scales emits CSS custom properties `--step--2`…`--step-5` via `clamp()`, with CSS/SCSS/PostCSS output and Viewport/Container targeting. A fluid-viewport ramp is a browser concept; the terminal projection (fixed cell) needs a non-fluid step→size mapping — that projection is a fonts.nix metrics row-group, not a `clamp()` egress. Source: https://utopia.fyi/type/calculator/.

## [05]-[TOKEN_BUILD_TOOLING]

Candidate rosters for a single-source→multi-egress token pipeline; each row carries capability delta + exact CLI/config surface. Adoption verdict: Forge already runs the single-source topology in Nix — these are egress-format references and DTCG-conformance checks, not a pipeline to import wholesale.

- [STYLE_DICTIONARY]: Transforms one token corpus into platform files through `source`, `include`, `tokens`, `expand`, `platforms`, `hooks`, `usesDtcg`, `transforms`, `transformGroup`, `buildPath`, `files`, `actions`, `destination`, `format`, `filter`, `options`, `outputReferences`. DTCG first-class since v4 (auto-detected via `usesDtcg`); `2025.10` support tracked as in-progress v5 work. Source: https://styledictionary.com/reference/config/, https://styledictionary.com/info/dtcg/.
- [TERRAZZO]: Consumes DTCG JSON, generates CSS, Sass, JS/TS, Swift, Tailwind, Storybook, and custom outputs. CLI `@terrazzo/cli`: `tz build`, `tz check [file]`, `tz lint` (alias of `check`), `tz init`, `tz lab`, `tz format`, `tz normalize` (alias of `format`), `tz import`; `--help`, `--version`; `DEBUG=*|parser:*|plugin:*`. Migration is the Cobalt→Terrazzo rename (Cobalt 2.0 became Terrazzo), with `2025.10` resolver support work ongoing — no `bundle` subcommand exists. Source: https://terrazzo.app/docs/, packages/cli/bin/cli.js.
- [TOKENS_STUDIO_SD_TRANSFORMS]: Normalizes Tokens Studio exports into Style Dictionary flows and aligns types to DTCG. `@tokens-studio/sd-transforms`, `register`, `expandTypesMap`, `preprocessors: ["tokens-studio"]`, `transformGroup: "tokens-studio"`, and the `ts/*` transforms (`ts/resolveMath`, `ts/size/px`, `ts/size/lineheight`, `ts/typography/fontWeight`, `ts/color/modifiers`, `ts/size/css/letterspacing`, `ts/color/css/hexrgba`, `ts/shadow/innerShadow`). Source: https://github.com/tokens-studio/sd-transforms.
- [COLOR_JS]: Parses/converts wide-gamut color and exposes `WCAG21`, `APCA`, `Lstar`, `DeltaPhi` contrast under one API — the contrast owner for a proof board that scores beyond WCAG. Source: https://colorjs.io/docs/contrast.html.
- [CULORI]: Function-oriented, tree-shakeable color library covering CSS Color Level 4 formats, extra color spaces, differences, interpolation, gradients, blend modes — the conversion owner for a palette→oklch/lab projection. Source: https://culorijs.org/.
- [APCA_W3]: Approved public-guideline APCA package (`apca-w3`); the canonical implementation behind any APCA score Forge emits. Source: https://github.com/Myndex/SAPC-APCA.

## [06]-[SELF_CONTAINED_HTML_PRIMITIVES]

The F07 dashboard renders under the html-studio contract; these are the browser primitives that cover its needs with zero external dependency.

- [ARTIFACT_CONTRACT]: An html-studio artifact is one self-contained file — `<!doctype html>`, one `<title>`, one `<style>`, one executable `<script>`, optional inert JSON payload scripts, zero CDN/webfont/remote-image dependency, dark+light rendering, headless gate/render validation. The inert JSON payload script is the seam where `palette.json`/register JSON embeds. Source: `.claude/skills/html-studio/SKILL.md:10`, `:36`, `:41`, `:42`.
- [COMPONENTS]: Custom elements, shadow DOM, `<template>`, `<slot>` via `customElements.define()`, `Element.attachShadow()` cover reusable dashboard tiles inside one file; CSS custom properties (`--*`) carry the theme cascade. Source: https://developer.mozilla.org/en-US/docs/Web/API/Web_components, https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/--*.
- [GRAPHICS]: `<canvas>`/`CanvasRenderingContext2D`/`Path2D` cover data-viz and animation; inline `<svg>` covers text-based vector markup interoperating with CSS/DOM/JS — charts and diagrams without a library. Source: https://developer.mozilla.org/en-US/docs/Web/API/Canvas_API, https://developer.mozilla.org/en-US/docs/Web/SVG.
- [INSPECTORS]: `<dialog>` with `showModal()`/`show()`/`close()`/`method="dialog"`, plus `popover`/`popovertarget`, cover modal and non-modal inspector surfaces for drill-down rows. Source: https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/dialog.

## [07]-[PATTERNS_WORTH_COMPOSING]

- [SINGLE_SOURCE_MULTI_EGRESS]: Style Dictionary and Terrazzo both drive many platform outputs from one token corpus through named build targets — the exact topology Forge already runs through `config.forge.theme` → `palette.json`, tmTheme, Lua, WezTerm, VS Code, base24 YAML, coverage ledger. The DTCG/tooling world is an egress-format vocabulary to project into, not an engine to import. Source: https://styledictionary.com/reference/config/, https://terrazzo.app/docs/, `modules/home/theme.nix:724`, `:831`.
- [DTCG_ADAPTER_OVER_COLOR_RECORD]: `mkColor` already emits `hex` + `{r,g,b}` + `rgba`; a DTCG color token is `hex` fallback + `colorSpace`/`components` + `alpha`, so a DTCG egress is a thin projection over the existing color record — the numeric renderings are already computed, only the `$value` envelope is new. Source: https://www.designtokens.org/tr/drafts/color/#color-type, `modules/home/theme.nix:18-31`.
- [EVAL_TIME_CONTRAST_ASSERTIONS]: WCAG 2.2 math is deterministic over sRGB hex; `base24Slots` enumerates every role hex; therefore per-role-pair contrast assertions are a pure Nix `let`-fold at eval, failing the build like `targetsProved` fails on a missing owner path. APCA layers an advisory score on the same pairs but never gates. Source: https://www.w3.org/TR/WCAG22/#dfn-contrast-ratio, `modules/home/theme.nix:453-460`, `:593`.
- [TOKENIZED_TYPE_RAMP]: DTCG types family/weight/dimension/`typography` + Utopia's named step vocabulary + fonts.nix metrics rows compose into a type ramp owned as fonts.nix metrics and projected as DTCG dimension/typography tokens per renderer. Source: https://www.designtokens.org/tr/drafts/format/#typography, https://utopia.fyi/type/calculator/, `modules/home/fonts.nix:105`, `:119`.
- [ONE_FILE_DASHBOARD]: Web Components + CSS variables + Canvas/SVG + `<dialog>` cover tiles, theme cascade, charts, and inspectors inside the single html-studio file; the inert JSON payload folds register/receipt JSON so agents read the same typed facts the page renders (F07 dual-consumer unlock). Source: `.claude/skills/html-studio/SKILL.md:41-42`, `docs/atlas/interconnection.md:64-72`.

## [08]-[GAPS]

The next research wave must hunt:

- DTCG `$extensions` vendor-namespacing convention — the exact key grammar Style Dictionary/Terrazzo read for custom data — to design the `palette.json`→DTCG adapter without losing `triple`/`csv`/glyph rows.
- APCA Lc threshold table (font-size × weight → required Lc) — the actual advisory lookup, since APCA scores rather than gates and no threshold is captured yet.
- Motion-token consumer reality: which of wezterm/zellij/yazi/nvim honor `duration`/`cubicBezier` at all — most terminals do not animate — to know the true projection surface for the motion axis before minting anticipatory rows.
- Per-surface density modeling: whether DTCG or any tooling treats "density" as a first-class type or as a dimension-token convention — F03 names density rows but no type owns it.
- Terminal type-ramp projection: the non-fluid step→fixed-cell-size mapping (Utopia's `clamp()` is viewport-fluid and browser-only), so the ramp reaches wezterm/zellij/nvim correctly.
- Smallest-closure DTCG consumer for a Nix build: whether Style Dictionary or Terrazzo can read Nix-generated JSON directly and which adds the least closure if a real egress consumer ever lands.
