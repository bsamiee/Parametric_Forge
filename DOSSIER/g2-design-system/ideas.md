# [G2_DESIGN_SYSTEM_IDEAS]

Wave-2 ideation for F03 (theme owner as design system) and F07 (estate dashboards). Section [01] closes the wave-1 GAPS with verified facts; every idea below cites a closure row or a `capabilities.md`/`ecosystem.md` row plus its estate anchor. An idea is thesis + verified capability + unlock + anchor — never a design the implementer must obey.

## [01]-[GAP_CLOSURES]

Verified facts closing the wave-1 GAPS; the TRUTH rows the ideas cite.

- [DTCG_EXTENSIONS]: `$extensions` keys are vendor-namespaced by reverse domain (spec SHOULD); payloads are arbitrary JSON; tools that process token files MUST preserve unrecognized `$extensions` through round-trip — a normative guarantee. Style Dictionary and Terrazzo both pass `$extensions` payloads through unaltered. The Forge carry shape is one `com.parametric-forge.<tool>` key holding `triple`/`csv`/glyph rows, losing nothing. Source: https://www.designtokens.org/tr/drafts/format/#extensions, https://styledictionary.com/info/dtcg/, https://terrazzo.app/docs/.
- [APCA_LC_ANCHORS]: canonical Lc anchors — `90` body-text preferred, `75` body-text minimum, `60` content-text minimum, `45` large/headline text, `30` absolute minimum for any readable text (spot/placeholder), `15` non-text minimum (borders, dividers, disabled). APCA is polarity-sensitive (dark-on-light vs light-on-dark score differently; compare on absolute value) and publishes a font-size×weight→minimum-Lc lookup table plus ARC Bronze conformance tiers — an advisory score ladder layered over the WCAG gate, never the gate. Source: https://myndex.com/APCA/, https://readtech.org/ARC/, https://github.com/Myndex/SAPC-APCA.
- [MOTION_CONSUMER_REALITY]: exactly two real motion-token consumers exist on the estate's app set. wezterm: `cursor_blink_rate` (ms) with `cursor_blink_ease_in`/`cursor_blink_ease_out` accepting named easings AND `{CubicBezier={x1,y1,x2,y2}}`, `visual_bell.fade_in_duration_ms`/`fade_out_duration_ms` with the same easing grammar, `animation_fps`/`max_fps`. neovim: only via `mini.animate` — per-action `duration` plus named easing families (polynomial/exponential timing functions, no bezier). zellij has NO animation surface (open feature request zellij#4429); yazi's Theme struct carries no animation field — a `[theme.animation]` block lands in the ignored custom bucket. Source: https://wezterm.org/config/lua/config/visual_bell.html, https://wezterm.org/config/lua/config/cursor_blink_rate.html, https://github.com/nvim-mini/mini.animate, https://github.com/zellij-org/zellij/issues/4429, https://github.com/sxyazi/yazi (Theme struct).
- [DENSITY_VERDICT]: DTCG `2025.10` has NO density type — the complete type set is `color`, `dimension`, `fontFamily`, `fontWeight`, `duration`, `cubicBezier`, `number` plus six composites. Density is a `dimension`-row convention (per-surface spacing steps); a custom `$type` would be non-conformant. Source: https://www.designtokens.org/tr/drafts/format/#types.
- [TYPE_RAMP_PROJECTION]: wezterm `font_size` is one global point scalar per config — a terminal grid renders one cell metric; no per-pane or per-run size exists. zellij, yazi, and nvim-TUI set no font size (host-terminal owned; only GUI frontends read `guifont`). Utopia's `--step-*` `clamp()` output is viewport-fluid CSS, browser-only. A terminal type ramp is therefore a DISCRETE per-surface point-size table — one row per independently sizable surface — never a fluid scale. Source: https://wezterm.org/config/lua/config/font_size.html, https://zellij.dev/documentation/, https://yazi-rs.github.io/docs/, https://utopia.fyi/type/calculator/.
- [SMALLEST_CLOSURE_CONSUMER]: style-dictionary@5.5.0 is pure JS with zero native binaries and reads DTCG JSON directly from a `source` file path with `usesDtcg`; `@terrazzo/cli` drags vite/esbuild/rollup per-architecture native binaries. If a DTCG egress consumer ever lands in the Nix build, Style Dictionary is the smaller closure. Source: https://www.npmjs.com/package/style-dictionary, https://www.npmjs.com/package/@terrazzo/cli, https://styledictionary.com/reference/config/.
- [STYLIX_PURE_ENROLLMENT]: `stylix.base16Scheme` is `oneOf [path lines attrs]`; a full attrset bypasses the image-derived palette entirely (image derivation fires only at option default). `stylix.autoEnable = false` flips every target default off, and per-target `stylix.targets.<name>.enable = true` re-admits selectively (`mkEnableTargetWith` ANDs global and per-target flags) — Stylix can drive enrollment for chosen targets while Forge keeps palette ownership and its own projections; no mutual exclusion. catppuccin/nix cannot: `flavor`/`accent` are enum-validated Catppuccin values and `sources` recursive-updates a fixed corpus. Source: https://raw.githubusercontent.com/nix-community/stylix/master/stylix/palette.nix, .../stylix/target.nix, https://raw.githubusercontent.com/catppuccin/nix/main/modules/global.nix.
- [BASE16_NIX_API]: `mkSchemeAttrs` coerces path | YAML string | attrset into a scheme value that is simultaneously an attrset of slots (`base00`…, mnemonic names, `withHashtag.<slot>`, `override`) and a functor over templates (`config.scheme templateRepo` or `{templateRepo; target;}` selecting mustache variations). Pure-Nix YAML parse since v2.0.0, `use-ifd = "never"` default, zero runtime closure. `mkBase24` fills `base10`–`base17` with fallbacks `base10/11←base00`, `base12←base08`, `base13←base0A`, `base14←base0B`, `base15←base0C`, `base16←base0D`, `base17←base0E`. Source: https://raw.githubusercontent.com/SenchoPens/base16.nix/main/lib/mk-scheme-attrs.nix, .../lib/colors.nix (mkBase24 at :25-34), .../lib/util.nix.
- [GLANCES_FEED_GRAMMAR]: Glances 4.5.x REST is `/api/4/<plugin>`, `/api/4/<plugin>/<field>`, `/api/4/all`, `/api/4/pluginslist`; Central Browser columns declare in `glances.conf` `[serverlist]` as `columns=<plugin>:<field>[:<key>],…` (defaults `system:hr_name,load:min5,cpu:total,mem:percent`); `--stdout-json plugin.field,…` selects dot-notation paths; `--enable-mcp` mounts an MCP SSE endpoint at `/mcp/sse` beside the web server exposing per-plugin stats resources. Source: https://glances.readthedocs.io/en/latest/api.html, .../quickstart.html, https://github.com/nicolargo/glances.
- [MONITOR_THEME_FORMATS]: btop themes are flat `theme[key]="#RRGGBB"` files (main_bg/main_fg/hi_fg/selected_bg/box colors/gradient start-mid-end triples) discovered via `--themes-dir` — not Base16/24-indexed; a palette-matched btop needs a bespoke projection. Glances has no palette file — curses color classes only. Source: https://github.com/aristocratos/btop#themes, https://glances.readthedocs.io.
- [TESSERA_STATE]: the WTF→Tessera rename is announced but incomplete — the repo is still `wtfutil/wtf`, last release `v0.50.0`; the tile schema is stable: each module carries `type`, `position {top,left,width,height}`, `refreshInterval`, `enabled` in `config.yml`. Source: https://github.com/wtfutil/wtf.
- [LOCAL_MOTION_LITERALS]: wezterm is already a LIVE motion consumer on this estate — `cursor_blink_rate = 250`, `max_fps = 120`, `animation_fps = 120` sit as raw literals outside any owner. No VS Code motion keys are set anywhere in the estate. glances and btop are not installed; `bottom` is the installed monitor and already a bound target row. Source: `modules/home/programs/apps/wezterm/default.nix:243`, `:288-289`, `modules/home/theme.nix:432`.

## [02]-[TOKEN_FAMILY_IDEAS]

- [MOTION_ROW_GROUP_WEZTERM_FIRST]: Thesis — the theme owner grows a motion row-group (blink cadence, feedback fade durations, one easing vocabulary) whose FIRST consumer already exists: wezterm's live `cursor_blink_rate = 250` literal folds into the owner, and `visual_bell` fade rows give `state.attention` a temporal dimension. Verified capability: wezterm accepts ms durations and CubicBezier easings ([MOTION_CONSUMER_REALITY]); DTCG owns the vocabulary as `duration`/`cubicBezier` ([capabilities §02]). Unlock: motion stops being an unowned literal; the two-consumer law (wezterm + mini.animate, nothing else) caps the family at real projection targets — zellij/yazi rows would be the anticipatory-rows defect F03 already outlaws. Anchor: `modules/home/theme.nix` owner, `modules/home/programs/apps/wezterm/default.nix:243`.
- [TYPE_RAMP_AS_SURFACE_TABLE]: Thesis — the type ramp lands in `fonts.nix` metrics as a discrete per-surface point-size table (terminal, editor, screenshot, HTML step vocabulary), one modular-scale fold at eval producing named steps, NOT a fluid clamp. Verified capability: only wezterm `font_size`, VS Code editor font settings, and carbon accept independent sizes — all three already `binds = fonts` rows in the targets registry; Utopia's step naming is the vocabulary, its `clamp()` egress is browser-only ([TYPE_RAMP_PROJECTION]). Unlock: `metrics.size = 13.0` becomes the base of a ratio-derived ramp instead of a lone scalar; HTML artifacts get the same steps as CSS custom properties. Anchor: `modules/home/fonts.nix:119-126`, `modules/home/theme.nix:416`, `:421`, `:431`.
- [ELEVATION_AS_SHADOW_FOR_HTML_ONLY]: Thesis — terminals already carry elevation as the surface ladder (`crust→background→surface→raised→overlay→selected`); a separate elevation family exists only as DTCG `shadow` composites whose first consumer is the F07 dashboard page and `palette.html`. Verified capability: the surface roles ladder is complete ([capabilities §01 LOCAL_OWNER_SURFACE]); DTCG `shadow` is a first-class composite ([capabilities §02 TYPE_SET]). Unlock: no duplicate elevation vocabulary — color-elevation stays roles, depth-elevation lands as shadow rows with the page as consumer. Anchor: `modules/home/theme.nix:76-83`, `:567`.
- [DENSITY_AS_DIMENSION_ROWS]: Thesis — density lands as per-surface `dimension` rows (spacing steps keyed to the type-ramp base), conformant with DTCG's type set since no density type exists; the first consumer is the F07 dashboard's CSS spacing scale, the only surface with verified spacing consumption today. Verified capability: [DENSITY_VERDICT]; CSS custom properties carry the cascade ([capabilities §06 COMPONENTS]). Unlock: density stops being a named-but-unowned F03 axis; terminal density rows wait until a real consumer key is verified. Anchor: `.claude/skills/html-studio/SKILL.md`, F03 charter density clause.

## [03]-[EVAL_TIME_PROOF_IDEAS]

- [WCAG_ROLE_MATRIX_AS_EVAL_GATE]: Thesis — the full role×landing contrast assertion set runs as a pure Nix fold at eval, failing the build exactly like `targetsProved` throws on a missing owner path. Verified capability: WCAG 2.2 math is deterministic over sRGB hex, every role resolves to a hex, and Forge already computes the identical math in `palette.html` JS ([capabilities §03 WCAG22_MATH]). Unlock: the F03 "WCAG proofs as flake checks" unlock with zero runtime — a palette retune that breaks `text.muted` on `surface.overlay` fails `nix flake check`. Anchor: `modules/home/theme.nix:455-460`, `:593`.
- [APCA_ADVISORY_COLUMN]: Thesis — `palette.html` and `coverage.json` grow an APCA Lc column beside each WCAG ratio, scored against the anchor ladder (body 75/90, content 60, headline 45, spot 30, non-text 15) so every role pair carries both the gate verdict and the readability score. Verified capability: [APCA_LC_ANCHORS]; the in-page JS already walks every role×landing pair. Unlock: retunes see readability regressions the 4.5:1 gate is blind to (WCAG passes dark-on-dark pairs APCA scores near zero); polarity sensitivity is visible because the page knows which member is the text. Anchor: `modules/home/theme.nix:593-640`, `forge-theme-proof`.
- [FONT_COUPLED_LC_TARGETS]: Thesis — the proof lane derives each text role's REQUIRED Lc from the estate's actual type table (the APCA font-size×weight lookup at `metrics.size = 13.0` and the ramp steps), not from generic thresholds — the first place the fonts owner and theme owner compose in one assertion. Verified capability: the APCA lookup table maps px×weight→minimum Lc ([APCA_LC_ANCHORS]); fonts.nix owns size and weight facts ([capabilities §01 FONT_OWNER]). Unlock: "readable at the size we actually render" replaces "generically AA"; a ramp change re-prices every contrast verdict automatically. Anchor: `modules/home/fonts.nix:119-126`, `modules/home/theme.nix:593`.

## [05]-[PROJECTION_ENGINE_IDEAS]

Verified capability: [STYLIX_PURE_ENROLLMENT]. Unlock: gap rows flip to `bound` without hand-writing five more projections; the registry verdict column measures exactly what Stylix closed. Anchor: `modules/home/theme.nix:443-447`, `:455-460`.
- [ONE_ROW_VARIANTS_FROM_TINTED]: Thesis — the F03 one-row-theme-variant unlock lands as `mkSchemeAttrs` over any tinted-theming base24 scheme: one attrset swap re-derives every projection, with the eval-time WCAG matrix ([03]) as the admission gate a candidate variant must pass before it ships. ## [GAPS]

The next research wave must hunt:

- Whether the estate nvim config carries `mini.animate` (or would admit it) — the second motion consumer is verified upstream but unverified locally; without it the motion family has exactly one consumer.
- wezterm density-relevant keys in the live estate config (`window_padding`, tab-bar spacing) — density rows for terminals need a verified consumer key before any dimension row lands beyond the HTML scale.
- The oklch computation route: pure-Nix oklab/oklch math cost versus a build-time culori derivation — no route is measured, and the color record's growth shape depends on it.
- Stylix target-by-target coverage for the five gap rows (does Stylix actually ship glow/zsh-plugin/macos-accent targets?) and the flake-input pin's eval cost — the enrollment idea is seam-verified but coverage-unverified.
- Glances package closure on nix-darwin (Python dependency tree) versus the receipts-only wave-1 payload — the admission price for the dual-consumer feed is unmeasured.
- Whether any tinted-theming tmTheme/sublime template approaches the scope fidelity of Forge's `syntaxScopes` pivot — decides if the tmTheme projection ever joins the mkSchemeAttrs collapse or stays owned forever.
- Tessera rename completion and whether the settled successor changes the tile schema the dashboard grammar borrows.
