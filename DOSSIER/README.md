# [DOSSIER]

Research corpora for the frontier groups: verified facts that ground each group's implementation pass — real configurations, current tool capabilities, extension and plugin candidates with their admission evidence, and patterns lifted from advanced OSS estates. A dossier carries TRUTH, never code suggestions and never designs: exact option spellings, capability inventories with sources, candidate rosters with capability deltas, and named patterns worth stealing. The implementing agent rules every design; a dossier that prescribes one is defective.

Each group directory accumulates through iterative waves. Wave 1 is GROUNDING: codex research lanes pile verified material into `capabilities.md` and `ecosystem.md`. Waves 2+ are IDEATION: codex lanes generate extension and integration ideas into `ideas.md` — per-app plugin/addon candidates for wezterm, yazi, zellij, and nvim researched in depth AND in relation to each other; richer cross-app integrations (ssh surfaces, floating-pane and pane-management concepts, new pane types, tunneling, genuinely novel directions); `modules/home/scripts` integration growth — every idea grounded in a verified capability, spanning small extensions to structural concepts, world-class per app and in concert. After each codex wave, one opus curation pass per group restructures for signal — trimming the unverifiable, merging duplicates, sharpening to the estate register, naming the gaps the next wave hunts. After the second codex→opus cycle, one fable grounding pass sweeps all groups: fixes nonsense and naivety, clarifies, extends the strong ideas, and opens new facets so later waves build on richer ground. Waves repeat until a group's curation adds nothing. Lane logs and run droppings (`.*.log`, stderr captures) are transient: each wave transition deletes the prior wave's noise so a group directory carries only its curated lens files. Group charters live in [docs/FRONTIER.md](../docs/FRONTIER.md); the group map (`g6-cross-app/` joins at wave 2 for concert-level ideas no single group owns):

- `g1-attention-receipts/` — F01 attention fabric, F02 receipts query plane, F10 pane capture, F11 zjstatus notifications
- `g2-design-system/` — F03 theme tokens (typography, color theory, motion), F07 estate dashboards
- `g3-vscode-flagship/` — F13 follow-ons (product icons, extras cull, workspace slimming, keybindings rail)
- `g4-session-fabric/` — F05 workspace-row unification, F12 VFS mounts
- `g5-language-harness/` — F06 language parity table, F08 harness kernel law

Consumption contract: a group implements as ONE fable 3-pass (initial → critique → red-team) that reads the group's dossier IN FULL from disk plus the FRONTIER charters, with the campaign doctrine (`docs/standards/`), seam map, and scars as binding law — rich deep passes, never incremental slices. Dossier facts are signals the implementer re-verifies at the anchor before building on them; a fact without a source row is a phantom the curation pass should already have deleted.
