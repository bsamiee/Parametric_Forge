# G4 Session Fabric Ideas

Extension and integration ideas for F05 (workspace-row unification) and F12 (VFS mounts), each grounded in a verified capability from `capabilities.md` / `ecosystem.md` or a live probe, carrying its source row. Idea shape: thesis, verified capability, unlock, estate anchor. The implementer rules every design; these are grounded directions, never prescriptions. Facts re-verify at the anchor before building.

## [01]-[GAP_CLOSURES_THAT_REDIRECT_IDEAS]

Live-probe verdicts that flip or harden the assumptions the wave-1 dossier left open. Each redirects at least one idea below.

- WezTerm packaged build is NIGHTLY `20260707-093716-fff02ca5`, installed as the `wezterm@nightly` cask, not stable. So `default_ssh_auth_sock` (nightly-only, per capabilities.md [03] `[ADDED]`) IS reachable — the wave-1 stable assumption is inverted. The 1Password socket can be pinned into the mux server so every mux-spawned pane and SSH domain inherits the correct agent. Source: probe `wezterm --version` → `20260707-093716-fff02ca5`; file:modules/darwin/homebrew/casks.nix:17; https://wezterm.org/config/lua/config/default_ssh_auth_sock.html.
- Ambient `SSH_AUTH_SOCK` is the macOS launchd agent (`/var/run/com.apple.launchd.*/Listeners`), NOT the 1Password socket. So Yazi's `identity_agent` default (SSH_AUTH_SOCK when absolute) would select the identity-less launchd agent; the explicit `identity_agent` 1Password row is load-bearing, not redundant. Same divergence hits the WezTerm mux without an explicit `default_ssh_auth_sock` pin. The identity-redundancy gap is CLOSED: keep the explicit row, and the mux needs its own explicit pin. Source: probe `echo $SSH_AUTH_SOCK`; comment at file:modules/home/programs/apps/yazi/default.nix:24-27.
- macFUSE and sshfs are ABSENT (no `/Library/Frameworks/macFUSE.framework`, no fuse kext, `sshfs` not on PATH); rclone IS present, Nix-provisioned at `/etc/profiles/per-user/bardiasamiee/bin/rclone` v1.74.3 on darwin 26.5. So any OS-mount lane is rclone-shaped and must avoid the macFUSE kext-approval step to stay agent-first (no host sudo). Source: probes `ls /Library/Frameworks/macFUSE.framework` (absent), `rclone version` (v1.74.3).
- Zellij v0.44.3 `list-sessions` emits TEXT only (`<name> [Created …ago]`, EXITED rows carry an `EXITED` marker); there is no JSON session API at the shell. Typed session state (`get_session_list`, `save_session`) is plugin-only. `delete-session [-f]` and `list-sessions -s|-r` are confirmed. So `forge-zellij-state/v2` resurrection state comes from parsing the EXITED marker already captured in `inventory()`, not a new typed API. Source: probes `zellij list-sessions --no-formatting`, `zellij delete-session --help`; file:modules/home/programs/apps/zellij/ops.nix:142-150.
- Estate serialization config is `session_serialization true` + `serialize_pane_viewport true` — and the v0.44.3 spelling is `serialize_pane_viewport`, correcting capabilities.md [02]'s `pane_viewport_serialization`. `scrollback_lines_to_serialize` defaults to full scrollback at `0`. Source: probes `zellij setup --dump-config | grep serializ`, file:modules/home/programs/apps/zellij/config.nix:393-394.
- The inventory grammar is already a 9-kind discriminated row graph — `session`, `session-exited`, `tab`, `pane`, `starred-pane`, `project`, `worktree`, `layout`, `macro`, `agent-lane` — each row `{row_id, kind, label, detail, target}`. `session-exited` is discoverable at the picker but the `state/v1` emitter drops the classification (emits raw split lines). Source: file:modules/home/programs/apps/zellij/ops.nix:139-203, ops.nix:249-256.

## [02]-[F05_WORKSPACE_ROW_VOCABULARY]

- Thesis: the workspace row absorbs the WezTerm SSH domain so a single row projects the picker entry, the `ssh_domains` row, the zellij session identity, and the layout together instead of the four hand-seams the charter names. Verified capability: `config.forge.ssh.hosts` already fans one tunnel row into WezTerm `ssh_domains` (`SSH:<host>`, remote_address/username/multiplexing) and Yazi VFS; the workspace rows are a separate naming table under the `estate-repo` domain. Unlock: a remote workspace is one row that names its host, and the SSH domain, VFS mount, layout, and acceptance row derive — no second table. Anchor: file:modules/home/programs/apps/wezterm/default.nix:46-66, ops.nix inventory graph. Source: capabilities.md [01].
- Thesis: promote the `session-exited` kind from a picker-only row to a first-class workspace-row state so a workspace's lifecycle (live / resurrectable / cold) is one enum on the same row, not a parallel discovery. Verified capability: `inventory()` already classifies `session-exited` via the EXITED marker; `forge-workspace --json` already joins registry rows to a `live` boolean from `wezterm cli list`. Unlock: `forge-workspace --json` carries `{live, resurrectable, cold}` per slug, so the picker, receipts, and acceptance read one lifecycle field. Anchor: ops.nix:145, file:modules/home/programs/apps/wezterm/default.nix:462-465. Source: capabilities.md [01], [02].
- Thesis: bind lifecycle hooks at the workspace row — creation, teardown, first-attach — mirroring Smug's `before_start`/`stop`/`attach_hook`/`detach_hook` and Sesh's `startup_command`. Verified capability: Smug binds exactly these four lifecycle points at the session row; the estate has no session-lifecycle hook today (forge-workspace spawns, never runs a hook). Unlock: a remote workspace warms its VFS mount on `before_start` and tears it down on `stop` from the same row that names it. Anchor: forge-workspace spawn path. Source: ecosystem.md [05] (Smug hooks), ecosystem.md [02].
- Thesis: split the workspace row into a compact discovery row plus a referenced layout row, delegating rich pane composition to the zellij layout engine (Sesh's `startup_command → tmuxinator start` split). Verified capability: Sesh's `default_session.startup_command` calls a layout engine and `session[].windows` references reusable named window rows; zellij layouts are already first-class KDL rows in the inventory graph (`kind: layout`). Unlock: the workspace row stays a one-line identity while its layout is a shared, named KDL asset reused across workspaces. Anchor: ops.nix layout rows (189-195), forge-workspace `--layout`. Source: ecosystem.md [05], capabilities.md [02].
- Thesis: wildcard workspace admission — a path-pattern row that admits any project directory as a workspace without an explicit registry entry, explicit rows winning over wildcard matches (Sesh `[[wildcard]]` order-sensitivity). Verified capability: `inventory()` already enumerates `kind: project` and `kind: worktree` rows by scanning `projectRootParents`; Sesh's wildcard model layers pattern rows under explicit rows. Unlock: every project root is a latent workspace with a default layout, and only divergent workspaces need an explicit row. Anchor: ops.nix:171-188 project/worktree scan. Source: ecosystem.md [05] (wildcard), capabilities.md [01].

## [03]-[F05_RESURRECTION_TO_RECEIPTS_PROJECTION]

- Thesis: emit `forge-zellij-state/v2` that classifies sessions into live vs resurrectable and joins the resurrectable set to the receipts plane, closing the charter's "session state joins the receipts plane" unlock. Verified capability: `inventory()` already parses the EXITED marker into `session-exited` rows; `state/v1` currently emits only raw `sessions` split lines (no classification); the estate's dual-receipt envelope (TSV + JSONL) is the sink. Unlock: a query like "which resurrectable sessions exist and when did each last serialize" is answerable from the receipts plane, not a live attach. Anchor: ops.nix:249-256 (state emitter), ops.nix:145 (exited parse). Source: capabilities.md [01] `[ADDED]`, [GAPS].
- Thesis: project the serialized-session cache as a receipts row — each session's last-serialization timestamp read from the cache directory mtime — so resurrectability carries freshness, not just existence. Verified capability: sessions serialize to `~/Library/Caches/org.Zellij-Contributors.Zellij/<uuid>/` every `serialization_interval` (default 1s); the cache dir is probe-confirmed present with per-session UUID subdirectories. Unlock: a resurrection picker row shows "cold since HH:MM" so a stale cold session is visibly distinct from a fresh one. Anchor: probe `ls ~/Library/Caches/org.Zellij-Contributors.Zellij/`; ops.nix inventory. Source: [01] probe, capabilities.md [02].
- Thesis: a session-freeze verb that snapshots a live session to a named, gate-validated layout asset — the estate's analog to Tmuxp `freeze` (the roster's only verified live→row snapshot), built on the existing `layout record` gate. Verified capability: `forge-zellij layout record` already dumps `action dump-layout`, gates the KDL through a disposable `attach --create-background` + `new-tab --layout` + `query-tab-names ≥2` parse check, and replaces the asset only on pass; Tmuxp `freeze` is the pattern donor, Zellij has no native session-freeze. Unlock: a workspace's live composition becomes a durable, reusable, gate-proven asset — the freeze half of Tmuxp's freeze/load round-trip. Anchor: ops.nix:328-358 (record gate). Source: ecosystem.md [05] `[ADDED]`, capabilities.md [02].
- Thesis: a receipts-driven resurrection-warming rail — `attach --create-background` pre-warms selected resurrectable sessions on session start so their first interactive attach is instant, each warm emitting a receipt. Verified capability: `zellij attach --create-background <session>` starts a session detached; the estate already uses it inside the layout-record gate, proving the spelling works headlessly. Unlock: cold workspaces named "always-warm" resurrect in the background at login, joining the live set before the operator asks. Anchor: ops.nix:341 (`attach --create-background` usage). Source: capabilities.md [02], ops.nix.

## [04]-[F05_FLOATING_PANE_AND_PANE_MANAGEMENT]

Verified capability: `deck.lua` stores float window ids in `wezterm.GLOBAL.deck_floats` keyed by `cmd.id`, focuses a live float before spawning, and synthesized floats (varying args) already spawn fresh without reuse; the registry rides `wezterm.GLOBAL` surviving reloads. Unlock: a per-workspace scratch/log/agent float that focuses-or-spawns scoped to the active workspace, not one shared window. Anchor: file:modules/home/programs/apps/wezterm/deck.lua:88-156. Source: capabilities.md [01].
- Thesis: a workspace-scoped float policy field on the workspace row — level, opacity, decorations, dimensions — so float appearance derives from the workspace rather than a global float table. Unlock: a remote/production workspace floats at a distinct level/opacity so the operator never confuses a prod float for a local one. Anchor: deck.lua:113-155, F05 charter (float policy). Source: capabilities.md [01], docs/FRONTIER.md:64.
- Thesis: a float-resurrection row that re-materializes named floats after a config reload or GUI restart from the receipts plane, since the `GLOBAL` registry dies on a full GUI exit. Unlock: after a WezTerm restart, the operator's standing floats (agent monitor, log tail) return from their last receipt rather than manual re-spawn. Anchor: deck.lua:142-155 (receipt on spawn), deck.lua:88-92 (GLOBAL registry). Source: capabilities.md [01].
- Thesis: a typed pane-capture float — a float that opens against a specific inner zellij pane's scrollback via `dump-screen --pane-id`, bridging the outer float rail to inner pane content. Verified capability: zellij exposes `action dump-screen --pane-id terminal_N --full [--ansi]` and `subscribe --pane-id … --format json`; the deck float rail spawns arbitrary `args`. Unlock: a float that captures and pins a named pane's output (an agent's last run) without leaving the workspace — pane capture (F10 adjacency) reaching the float surface. Anchor: capabilities.md [02] capture spellings, deck.lua spawn_float. Source: capabilities.md [02], [03].

## [05]-[F12_MOUNT_POLICY_AND_RCLONE_LANE]

- Thesis: the SSH-host row grows mount-policy fields — `paths`, `read_only` — that project into BOTH the Yazi VFS row and, for consumers needing POSIX paths, an rclone `nfsmount` lane that is macFUSE-free and sudo-free, keeping F12 single-source and agent-first. Verified capability: Yazi `[services.<host>]` already projects from `config.forge.ssh.hosts`; `rclone nfsmount remote:path /local` spins an internal userspace NFS server and mounts it via macOS's built-in `mount -t nfs` client — NO kernel extension, NO kext approval, NO host sudo — and on macOS a mount without `--vfs-cache-mode` is READ-ONLY by default, matching a read-only remote design exactly. Unlock: maghz volumes reachable by both Yazi (`sftp://maghz`) and POSIX-path tools (grep, editors) from one row set, with zero kext ceremony. Anchor: file:modules/home/programs/apps/yazi/default.nix:348-360, F12 charter. Source: https://rclone.org/commands/rclone_nfsmount/, https://rclone.org/commands/rclone_serve_nfs/, ecosystem.md [03].
- Thesis: the `cache posture` field the charter names has NO Yazi sink (`ServiceSftp` exposes no cache knob) and MUST land on the rclone lane, whose `--vfs-cache-mode` is the only verified cache-policy sink. Verified capability: `ServiceSftp` has no `cache` field (capabilities.md [04] `[ADDED]`); rclone models cache posture as `--vfs-cache-mode off|minimal|writes|full` plus `--dir-cache-time`/`--poll-interval`. Unlock: cache posture becomes a real, projectable field owned by the rclone lane instead of a dead schema row on the Yazi side. Anchor: capabilities.md [04] `[ADDED]`, ecosystem.md [03] `[ADDED]`. Source: capabilities.md [04], ecosystem.md [03].
- Thesis: the rclone lane stays CONFIG-FILE-FREE and reuses the SAME `config.forge.ssh.hosts` row source — one identity source, three consumers — via an on-the-fly connection string plus env vars, never a stateful `rclone.conf`. Verified capability: rclone defines a remote entirely from `:sftp,host=…,user=…:path` connection strings plus `RCLONE_SFTP_HOST|USER|PORT|KEY_USE_AGENT` (and the `RCLONE_CONFIG_<REMOTE>_<KEY>` scheme), and `key_use_agent = true` / `RCLONE_SFTP_KEY_USE_AGENT=true` sources identity from a running ssh-agent (the 1Password socket) with no on-disk key. Unlock: a second VPS becomes one host row that Yazi, WezTerm, and rclone all fold, projected as env/connection-string rows with zero config file — the F12 "second VPS mounts with zero yazi edits" unlock extended to a stateless mount lane. Anchor: file:modules/home/programs/apps/yazi/default.nix:348-360 (single-source row pattern). Source: https://rclone.org/docs/, https://rclone.org/sftp/, ecosystem.md [03].
- Thesis: `forge-workspace` gains remote-workspace entries whose `cwd` is a mounted remote path riding the same host identity, so a remote workspace is a workspace-row that names its host and mount. Verified capability: `forge-workspace` today resolves only a local `cwd` from the naming rows and spawns a slug-named zellij session; the charter names "remote-workspace entries riding the same identity." Unlock: selecting a remote workspace attaches a zellij session whose cwd is the mounted maghz volume, no separate remote-open step. Anchor: file:modules/home/programs/apps/wezterm/default.nix:481-518 (forge-workspace resolve/spawn), F12 charter. Source: capabilities.md [01], docs/FRONTIER.md:71.

## [GAPS]

- [CLOSED] rclone macFUSE-free mount: `rclone nfsmount remote:path /mount` uses an internal userspace NFS server + macOS native `mount -t nfs` — no kext, no approval, no sudo; a mount without `--vfs-cache-mode` is read-only by default. Viable agent-first. Source: https://rclone.org/commands/rclone_nfsmount/.
- [CLOSED] rclone declarative config: `:sftp,host=…,user=…:path` connection strings + `RCLONE_SFTP_*` / `RCLONE_CONFIG_<REMOTE>_<KEY>` env vars fully specify a remote with ZERO rclone.conf — stateless-config in the Nix estate. Source: https://rclone.org/docs/.
- [CLOSED] rclone SFTP agent auth: `key_use_agent = true` (`RCLONE_SFTP_KEY_USE_AGENT`) sources identity from a running ssh-agent with no on-disk key; the single-identity-source thesis holds for rclone. Caveat: rclone docs describe "contact an ssh-agent" but never name `SSH_AUTH_SOCK`, so the 1Password-socket-via-`SSH_AUTH_SOCK` binding is convention, not doc-stated — re-verify the socket the rclone process actually reads at the anchor. Source: https://rclone.org/sftp/.
- Consumer ruling (OPEN): enumerate which estate consumers actually need POSIX mountpoint paths (non-Yazi tools) versus Yazi's in-process `sftp://` — that decides whether the rclone `nfsmount` lane joins the fabric or F12 stays Yazi-only with `paths`/`read_only` as Yazi-scoped fields.
- launchd mount lifecycle (OPEN): rclone `--daemon` works on macOS and SIGTERM unmounts, but rclone publishes NO launchd plist example; rule whether a `com.parametric-forge.<name>` LaunchAgent owns the process directly (no `--daemon`, launchd owns lifecycle) or manages a `--daemon` child, and how mount absence degrades under the estate's explicit-degrade-receipt law. Source: https://rclone.org/commands/rclone_mount/.
- resurrect.wezterm delta (OPEN): whether it adds anything over `forge-workspace` + Zellij resurrection + deck.lua float persistence, or is fully subsumed — decides if a WezTerm-side session package ever joins.
- WezTerm mux SSH_AUTH_SOCK pin (OPEN): now that the packaged build is nightly, confirm `default_ssh_auth_sock` actually propagates the 1Password socket to mux-spawned panes and SSH domains at the anchor (probe the socket a spawned pane sees), since the ambient default is the wrong launchd agent.
