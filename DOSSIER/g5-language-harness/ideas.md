## [01]-[PARITY_REGISTRY_AND_EVAL_CHECK]

The parity-table check design space the verified surfaces admit.

**Promote the fifth cell from runtime probe to eval-time flake check.** `health.lua` proves completeness only after nvim launches; a missing cell should fail `forge-redeploy --check-only` before a switch. Capability: the module system's `assertions` option (list of `{assertion, message}`) aborts toplevel evaluation and prints `message`, and `lib.asserts.assertMsg :: Bool -> String -> Bool` throws at eval; both fire when the home-manager/nix-darwin toplevel is built, and `nix flake check` builds every `checks.<system>.<name>` derivation. Verified: `lib/asserts.nix`, nix3-flake-check manual, `flake-modules/qa.nix:58`. Unlock: parity completeness gates the deploy rail, not a post-launch `:checkhealth`. Anchor: `flake-modules/qa.nix:58` checks attrset, `forge/health.lua:38` as the assertion vocabulary to lift into Nix.

**One `languages` registry owns all five cells and projects into every consumer.** Today a language spans four surfaces in two files (server row `default.nix:90`, grammar list `default.nix:23`, `fmt` lane `fmt.nix:21`, lint row `default.nix:350`) plus the marketplace. Capability: Helix's `languages.toml` carries file-types, roots, language-servers, grammar, formatter, and code-actions in ONE record with built-in/user/project override layers (verified <https://raw.githubusercontent.com/helix-editor/helix/master/book/src/languages.md>); the estate already projects one `servers` row family into two consumers via `lib.mapAttrs'` (`default.nix:428`). Unlock: F06's core promise — a new language is one parity row, and editor, harness, formatter, and marketplace derive from it. Anchor: `default.nix:90` servers as the row family that absorbs the other cells.

**The registry projects INTO treefmt AND `fmt` — treefmt structurally cannot be the sole owner.** Capability: treefmt-nix routes by filename glob ONLY (no shebang, no content dispatch) and exposes no self-test or invariant API (verified `treefmt-nix/module-options.nix` + README, `treefmt/README.md`); the `fmt` CLI owns shebang probing (extensionless executables and `#!` lines route to `shell`/`python`, `fmt.nix:63`, `fmt.nix:106`) and eight `--self-test` invariants — lane completeness, write/check head-token agreement, lockfile denial, SQL-dialect routing, jq compile gating (`fmt.nix:133`). Unlock: one registry keeps the two Nix formatter surfaces coherent while preserving the shebang+invariant capability treefmt cannot express, so neither surface is a migration target for the other. Anchor: `flake-modules/tooling.nix:68` treefmt block, `fmt.nix:106` `_lane_for`.

**Extend the marketplace-parity identity check to the other four cells.** The Claude parity section is the only one that proves IDENTITY (not mere presence): it cross-checks generated rows against the tracked `.lsp.json` AND the installed cache Claude actually loads. Capability: `identity_match` compares command/args/extensionToLanguage/settings across `claude-marketplace.json`, `.claude/lsp-marketplace/<plugin>/.lsp.json`, and `~/.claude/plugins/installed_plugins.json` (`health.lua:31`, `health.lua:129`). Unlock: grammar/fmt/lint cells gain drift detection, not just binary-present checks — a formatter whose config diverges from the generated law fails, matching the marketplace cell's rigor. Anchor: `health.lua:31` identity_match, `default.nix:428` marketplace projection.

**Type grammar-only rows as a distinct highlight cell so completeness never demands tools they will never carry.** The bundle carries 32 grammars against 8 LSP servers; `kdl`, `mermaid`, `json5`, `xml`, `regex`, `csv`, `diff`, `git_config`/`git_rebase`/`gitcommit`, `jsdoc`, `query`, `vim`, `vimdoc`, `html` are highlight-only with no formatter or LSP. Capability: the grammar list (`default.nix:23`) and `vim.treesitter`'s own healthcheck (`:checkhealth vim.treesitter`, verified health.txt) prove parsers independently of language tooling. Unlock: the parity assertion distinguishes full language rows (five cells required) from highlight rows (grammar only), so `mermaid` is not a false parity hole. Anchor: `default.nix:23` grammars, `health.lua:51` parser section.

## [02]-[FORMATTER_TRINITY_CONVERGENCE]

Closing the editor-vs-CLI web-formatter divergence — richer than the dossier framed it, because three surfaces exist and biome is already the law in two.

**Biome is canonical; conform is the sole outlier — flip the editor's web lane to biome.** Capability: treefmt `programs.biome` carries `biome.json`'s bytes verbatim and is THE `formatting` flake check (`tooling.nix:80`, `qa.nix:61`); the `fmt` `web` lane routes `.ts/.js/.json/.css` to biome (`fmt.nix:25`); only conform routes the same extensions to prettier (`default.nix:346`); Biome formats JS/TS/JSX/TSX/JSON/JSONC/CSS/GraphQL at stable (verified <https://biomejs.dev/internals/language-support/>). Unlock: the editor stops emitting diffs the flake check rejects, and `biome.json` becomes the single web law across all three formatter surfaces plus the VSCode extension (already biome-owned, `tooling.nix:76`). Anchor: `default.nix:346` conform format facts.

**Prose stays prettier in all three surfaces — and markdown formatting is currently ungated.** Capability: Biome markdown is in-progress, not stable (parse+format experimental, no lint) per biomejs.dev language-support, so prettier remains required for `.md`; treefmt carries NO prettier/prose row (`tooling.nix:74` programs list), meaning `.md` formatting ships unenforced by `nix flake check` while `fmt` (`fmt.nix:26`) and conform both format it. Unlock: name prose as a deliberate prettier-only cell AND close the flake-check hole — markdown drift currently reaches `main` unblocked. Anchor: `fmt.nix:26` prose lane, `tooling.nix:74` treefmt programs, `qa.nix:61`.

**HTML is a movable cell, not a divergence — biome now formats it opt-in.** Capability: `.html` rides `prose`→prettier today (`fmt.nix:48`), but Biome HTML formatting is supported behind an opt-in flag (verified biomejs.dev language-support). Unlock: consolidating `.html` under biome shrinks the prettier surface toward markdown-only, tightening the single-web-law story. Anchor: `fmt.nix:48` `.html`→prose, `default.nix:346`.

**Close the flake-check coverage gap: seven of fourteen fmt lanes ship ungated.** Capability: treefmt gates alejandra/biome/ruff/yamlfmt/shfmt/sqruff(pg+duckdb)/stylua (`tooling.nix:74`); `fmt` owns fourteen lanes including toml(taplo)/csharp(csharpier)/swift/osa/prose/jq (`fmt.nix:21`); the jq lane is ALREADY flake-check-gated by a fmt-driven derivation running `fmt --self-test` + `fmt --check` (`qa.nix:69`). Unlock: the `jq-syntax` check is the reusable template — a fmt-driven `formatting` check could subsume treefmt entirely, or treefmt gains taplo/csharpier rows, closing parity between what `fmt` formats and what the gate enforces. Anchor: `qa.nix:69` jq-syntax as the pattern, `fmt.nix:21` lanes.

**Biome lint fills the empty TypeScript lint cell — one tool, both cells.** Capability: Biome ships a linter distinct from its formatter — `biome lint`/`biome check`, 509 rules over JS/TS/CSS/JSON/GraphQL/HTML (verified <https://biomejs.dev/linter/>); the biome binary is already installed for formatting (node owner, `capabilities.md` §05), yet conform's `lint.ft` carries no typescript row (`default.nix:350`). Unlock: TS lint parity with zero new tooling. Anchor: `default.nix:350` lint.ft.

**luacheck lint cell is a one-row fill.** Capability: the lua owner installs luacheck (`lua-tools.nix:17`, `capabilities.md` §05) but `lint.ft` has no lua row (`default.nix:350`). Unlock: lua lint parity from an already-installed linter. Anchor: `default.nix:350`.

## [03]-[SETUP_ENV_DECOMPOSITION]

F08's "row-driven resolver folds with dual receipts and shape-asserted admissions" against the live 449-line `setup-env.sh` — its fold boundaries and receipt shapes already exist embryonically.

**The `DOPPLER_SOURCES` row IS the resolver fold; F08 promotes the colon-string to a shape-asserted admission.** Capability: `_load_secrets` fans out parallel `_resolve_source` workers over rows shaped `project:config:snapshot[:TOKEN_ENV_VAR]`, each emitting `outcome|nkeys|age|auth|reason` meta (`setup-env.sh:39`, `setup-env.sh:115`, `setup-env.sh:164`); the key admission is already shape-gated (`[[ "${k}" =~ ^[A-Za-z_][A-Za-z0-9_]*$ ]]`, `setup-env.sh:228`). Unlock: asserting field count and token-var naming at parse makes a malformed source row fail loud rather than silently mis-resolve — the "shape-asserted admissions" clause realized on the row that already drives the fold. Anchor: `setup-env.sh:39` DOPPLER_SOURCES, `setup-env.sh:120` IFS parse.

**Dual receipts already exist — `_RECEIPT` (file+stderr) and `_ALERTS` (stdout under degraded state); formalize the split as kernel-law receipt shape.** Capability: `_RECEIPT` rows carry `ok`/`warn`/`DEAD`/`note` to `SESSION_CACHE_DIR/receipt` and stderr; `_ALERTS` reach stdout only when a source degrades; both publish key NAMES only via `jq -r keys_unsorted` (`setup-env.sh:156`, `setup-env.sh:353`, `setup-env.sh:364`). Unlock: the receipt/alert split becomes the reusable dual-receipt contract every `.claude/` script adopts under the unification mandate. Anchor: `setup-env.sh:353` publish block.

**The three-arm `_classify` verdict (`live`/`snapshot`/`dead`) is a reusable resolver vocabulary.** Capability: `_classify` maps return code, offline state, and snapshot mtime-advance to a verdict — the mtime advance IS the live proof, since `--no-cache` forces a full fetch and a cache-served read would fake liveness (`setup-env.sh:89`, `setup-env.sh:104`). Unlock: a named verdict vocabulary other resolver-shaped `.claude/` scripts fold into instead of reinventing liveness detection. Anchor: `setup-env.sh:104` `_classify`.

**The three runtime lanes single-flighted through a lock dir are the shell-side determinism law F08 audits.** Capability: warm replay + detached `--refresh`, cold inline single-flight via `mkdir` lock with >3-minute stale-takeover (`setup-env.sh:386`, `setup-env.sh:422`, `setup-env.sh:435`); the workflow determinism gate (`validate-workflow.mjs` banning `Date.now`/`Math.random`) has no shell analogue (`capabilities.md` §06). Unlock: the lane/lock discipline is the pattern a shell-script determinism audit enforces across `.claude/`, extending F08's determinism law past JS workflows. Anchor: `setup-env.sh:386` refresh lane, `validate-workflow.mjs` (capabilities §06).

**The kernel binds the correct event — expanding env writes to Setup/CwdChanged is unsupported per current docs (corrects `capabilities.md:77`).** Capability: verified this session, only `SessionStart` hooks receive `CLAUDE_ENV_FILE` (variables written there flow to all later Bash); the dossier's claim that `SessionStart/Setup/CwdChanged/FileChanged` all write it is contradicted by the live docs. `SessionStart` admits only `command` and `mcp_tool` handlers of the 30-event surface (verified <https://code.claude.com/docs/en/hooks>). Unlock: F08's kernel law pins the env-write event to `SessionStart` alone and drops the four-event assumption. Anchor: `.claude/settings.json:19` SessionStart binding.
